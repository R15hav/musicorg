# CLI Reference

Every command follows the form:

```bash
musicorg [GLOBAL OPTIONS] <command> [COMMAND OPTIONS]
```

Running `musicorg` with no subcommand launches the [wizard](wizard.md).

Source: [`src/musicorg_cli/main.py`](https://github.com/R15hav/musicorg/blob/main/src/musicorg_cli/main.py)

---

## Global options

These apply to every command:

| Option | Description |
|---|---|
| `--library TEXT` | Library slug or name (sets the state directory) |
| `--state-dir PATH` | Override the state directory path directly |
| `--config PATH` | Override the config file path |
| `--quiet` | Suppress informational output |
| `--verbose` | Verbose output |
| `--json` | Emit JSON progress lines to stdout (for piping) |

## File-tree group (Stage 1)

### `organize`

```bash
musicorg organize
```

Alias for the guided wizard. Equivalent to running `musicorg` with no subcommand.

---

### `scan`

```bash
musicorg --library <name> scan PATH [--exclude PATH]...
```

Walk `PATH` for audio files, read tags via mutagen → ffprobe → mediainfo cascade, compute per-file audio-stream SHA-256 fingerprints.

**Writes:** `<state>/01_tags.csv`

| Option | Default | Description |
|---|---|---|
| `PATH` | *(required)* | Root directory to walk |
| `--exclude PATH` | — | Subtree to skip. Repeatable. |

Exit codes: `2` = no audio files found; `3` = more than 5% of files unreadable.

---

### `dedupe`

```bash
musicorg --library <name> dedupe [--interactive]
```

Group duplicates by normalised `title | artist` + duration bucket. Pick winners by bitrate and filename junk score.

**Reads:** `<state>/01_tags.csv`
**Writes:** `<state>/07_winners.csv`, `<state>/07_duplicates.csv`, `<state>/07_groups.csv`

| Option | Default | Description |
|---|---|---|
| `--interactive` | off | Launch the [two-pane dup-review TUI](tui.md#dupreviewapp) |

---

### `resolve`

```bash
musicorg --library <name> resolve
```

Merge folder name, embedded tags, and filename evidence into one canonical record per winner. Apply country heuristics.

**Reads:** `<state>/07_winners.csv`
**Writes:** `<state>/08_resolved.csv`

---

### `plan`

```bash
musicorg --library <name> plan [--overrides PATH]
```

Compute target destination paths. Albums with a single track are demoted to `Singles/`.

**Reads:** `<state>/08_resolved.csv`
**Writes:** `<state>/09_plan.csv`

| Option | Default | Description |
|---|---|---|
| `--overrides` | — | Path to a `manual_fills.json` for manual destination overrides |

---

### `apply`

```bash
musicorg --library <name> apply [--mode move|copy|symlink] [--dest PATH] [--dry-run]
```

Execute `09_plan.csv`. Quarantines dup losers to `_duplicates/`. Writes an undo shell script.

**Reads:** `<state>/09_plan.csv`, `<state>/07_duplicates.csv`
**Writes:** `<state>/undo_<TS>.sh`

| Option | Default | Description |
|---|---|---|
| `--mode` | `move` | `move`, `copy`, or `symlink` |
| `--dest` | — | Required when `--mode copy` |
| `--dry-run` | off | Preview without moving files |

Exit code `4` = one or more source files were missing.

---

### `quarantine`

```bash
musicorg --library <name> quarantine
```

Move dup losers recorded in `07_duplicates.csv` to `<music_root>/Music/_duplicates/<original-subpath>/`. Safe to run without a `09_plan.csv` in place.

---

### `misc-sweep`

```bash
musicorg --library <name> misc-sweep [--include-images] [--include-text] [--dry-run]
```

Move non-audio top-level files to `<music_root>/Music/_misc/`.

| Option | Default | Description |
|---|---|---|
| `--include-images` | off | Also sweep image files |
| `--include-text` | off | Also sweep plain-text files |
| `--dry-run` | off | Preview without moving |

---

### `zip-probe`

```bash
musicorg --library <name> zip-probe [--quarantine-duplicates]
```

Diff the contents of every `.zip` in the music tree against the organised filenames. Useful for detecting zip backups of files that are already organised.

| Option | Default | Description |
|---|---|---|
| `--quarantine-duplicates` | off | Move zip-duplicates to `_duplicates/` |

---

### `undo`

```bash
musicorg --library <name> undo [--list] [--script PATH]
```

Run a file-move undo script generated by a previous `apply`.

| Option | Default | Description |
|---|---|---|
| `--list` | off | List available undo scripts without running one |
| `--script PATH` | — | Run a specific script. Defaults to the most-recent one. |

---

## Canonical metadata group (Stage 2)

### `canonicalize`

```bash
musicorg --library <name> canonicalize [--tiers TEXT] [--skip-tier TEXT] [--only-decisions TEXT]
```

Run the tiered lookup chain (iTunes → JioSaavn → Shazam) against every audio file in the organised tree.

**Reads:** files under `<library_root>/Music/` (excludes `_duplicates`, `_misc`, `_replaced`, `_upgrade_staging`)
**Writes:** `<state>/16_merged.csv`

| Option | Default | Description |
|---|---|---|
| `--tiers` | `itunes,jiosaavn,shazam` | Comma-separated tier list |
| `--skip-tier` | — | Remove one tier from the list |
| `--only-decisions` | — | Filter to specific decision buckets |

Requires `musicorg apply` to have been run first (needs the organised tree).

---

### `review`

```bash
musicorg --library <name> review [--export] [--out PATH] [--import PATH] [--interactive]
```

Round-trip a CSV through the user for manual approval, or launch a TUI.

**Reads:** `<state>/16_merged.csv`
**Writes (--export):** `<state>/19_review.csv`
**Writes (--import):** `<state>/19_approvals.json`

| Option | Default | Description |
|---|---|---|
| `--export` | off | Write `19_review.csv` for manual editing |
| `--out PATH` | `19_review.csv` | Override output path for `--export` |
| `--import PATH` | — | Path to user-edited review CSV; validates and emits `19_approvals.json` |
| `--interactive` | off | Launch [CanonicalReviewApp TUI](tui.md#canonicalreviewapp) if `16_merged.csv` exists, else [PlanReviewApp TUI](tui.md#planreviewapp) |

The `approve` column in the review CSV accepts: `itunes`, `jiosaavn`, `shazam`, `manual`, `skip`.

---

### `approve`

```bash
musicorg --library <name> approve --rule "jiosaavn>shazam>itunes" [--confidence-min FLOAT] [--bucket TEXT]
```

Non-interactive batch approval. Applies a tier-priority rule to unresolved rows.

**Reads:** `<state>/16_merged.csv`
**Writes:** `<state>/19_approvals.json`

| Option | Default | Description |
|---|---|---|
| `--rule` | *(required)* | Priority string, e.g. `"jiosaavn>shazam>itunes"` |
| `--confidence-min` | `0.65` | Minimum confidence to approve |
| `--bucket` | `review,low` | Which decision buckets to process |

---

### `canonical-apply`

```bash
musicorg --library <name> canonical-apply [--dry-run] [--no-rename] [--include-low]
```

Take a tag snapshot, write tags, and optionally rename files for `auto_apply` rows and any recorded approvals.

**Reads:** `<state>/16_merged.csv`, `<state>/19_approvals.json` (if present)
**Writes:** `<state>/17_dryrun_diff.csv`, `<state>/backups/tag_snapshot_<TS>.json`, `<state>/undo_phase18_<TS>.py`; when `19_approvals.json` is present: `<state>/backups/tag_snapshot_phase20_<TS>.json`, `<state>/undo_phase20_<TS>.py`

| Option | Default | Description |
|---|---|---|
| `--dry-run` | off | Preview without writing |
| `--no-rename` | off | Write tags without renaming files |
| `--include-low` | off | Also apply `low`-confidence rows |

---

### `canonical-undo`

```bash
musicorg --library <name> canonical-undo [--list] [--latest] [--snapshot TS]
```

Restore tags and filenames from a previously generated tag snapshot.

| Option | Default | Description |
|---|---|---|
| `--list` | off | List available snapshots |
| `--latest` | off | Restore from the most-recent snapshot |
| `--snapshot TS` | — | Restore from a specific snapshot by timestamp |

## Upgrade group (Stage 3)

### `refingerprint`

```bash
musicorg --library <name> refingerprint [--lossy-only/--all-files] [--path PATH]
```

Run a fresh Shazam audio-fingerprint pass. Harvests Apple Music URLs and writes `30_shazam_refingerprint.csv`.

**Writes:** `<state>/30_shazam_refingerprint.csv`

| Option | Default | Description |
|---|---|---|
| `--lossy-only` / `--all-files` | `--lossy-only` | Restrict to lossy formats (mp3, aac, m4a non-ALAC) |
| `--path PATH` | — | Limit to files under this path |

Aborts early if the Shazam circuit breaker is tripped. See [troubleshooting](troubleshooting.md#shazam-circuit-breaker).

---

### `upgrade`

```bash
musicorg --library <name> upgrade [--dry-run] [--cookies PATH] [--wvd PATH] [--staging-dir PATH]
```

Drive `gamdl` to download ALAC versions of candidates with a resolved `apple_music_url`. Verifies each download with ffprobe.

**Reads:** `<state>/30_shazam_refingerprint.csv` (preferred) or `<state>/16_merged.csv`
**Writes:** `<state>/upgrade_skips.csv`, `<state>/_upgrade_staging/<run-id>/`

| Option | Default | Description |
|---|---|---|
| `--dry-run` | off | Preview without downloading |
| `--cookies PATH` | — | Path to `cookies.txt` |
| `--wvd PATH` | — | Path to Widevine device file (`.wvd`) |
| `--staging-dir PATH` | auto | Override staging root (default: `<state>/_upgrade_staging/`) |

Each run gets a unique staging subdirectory (`run_<TS>/`) so gamdl cannot silently skip re-downloads. See [upgrade.md](upgrade.md).

---

### `recover-staging`

```bash
musicorg --library <name> recover-staging [--staging-dir PATH] [--dry-run]
```

Recover ALAC files orphaned by a failed or interrupted upgrade run. Scans `_upgrade_staging/` and moves valid ALAC files to their correct destinations.

| Option | Default | Description |
|---|---|---|
| `--staging-dir PATH` | — | Override staging root |
| `--dry-run` | off | Preview without moving |

---

### `permanent-skip-report`

```bash
musicorg --library <name> permanent-skip-report [--format table|csv|json]
```

Show all permanently-skipped files grouped by reason.

**Reads:** `<state>/upgrade_skips.csv`

| Option | Default | Description |
|---|---|---|
| `--format` | `table` | Output format: `table`, `csv`, or `json` |

---

## TUI commands

### `fill`

```bash
musicorg --library <name> fill
```

Launch the [FillUnknownsApp TUI](tui.md#fillunknownsapp) — a per-row card editor for files with non-empty `needs_lookup` fields. Requires `08_resolved.csv`.

---

### `dedupe --interactive`

```bash
musicorg --library <name> dedupe --interactive
```

Launch the [DupReviewApp TUI](tui.md#dupreviewapp) — a two-pane duplicate picker.

---

### `review --interactive`

```bash
musicorg --library <name> review --interactive
```

Launch [CanonicalReviewApp](tui.md#canonicalreviewapp) if `16_merged.csv` is present, or [PlanReviewApp](tui.md#planreviewapp) if only `09_plan.csv` is available.

---

## Config commands

### `config init`

```bash
musicorg config init
```

Interactive setup for the global config file (`~/.config/musicorg/config.ini`). Prompts for AcoustID API key, MusicBrainz contact email, and default country.

---

### `config set`

```bash
musicorg config set <section.key> <value>
```

Set one config key. Key format is `section.key` (e.g. `acoustid.api_key`, `defaults.country`, `shazam.breaker_threshold`).

---

### `config show`

```bash
musicorg config show
```

Display the effective config for the current library (merged global + per-library).
